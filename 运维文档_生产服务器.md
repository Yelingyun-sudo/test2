## 四、定期清理任务（关键！每周必做）

### ⚠️ 生产服务器磁盘清理（高优先级）

**当前磁盘使用率 79%，必须定期清理以下项目：**

### 4.1 清理 Playwright 临时文件（⚠️ 最占空间）

**检查临时文件数量和大小**：
```bash
cd /tmp
echo "=== Playwright 临时文件统计 ==="
ls -d playwright_chromiumdev_profile-* 2>/dev/null | wc -l
du -ch playwright_chromiumdev_profile-* 2>/dev/null | tail -1
echo "================================"
```

**清理命令**（每周执行 2-3 次）：
```bash
cd /tmp

# 删除所有 playwright_chromiumdev_profile 目录
rm -rf playwright_chromiumdev_profile-*

# 同时清理其他 Playwright 相关临时文件
rm -rf playwright-artifacts-*
rm -rf .com.google.Chrome.*

# 验证清理结果
echo "清理后剩余："
ls -d playwright_chromiumdev_profile-* 2>/dev/null | wc -l
```

**清理后检查磁盘空间**：
```bash
df -h /
```

---

### 4.2 清理项目日志

**查看日志目录大小**：
```bash
du -sh /opt/website_analytics/backend/logs/
```

**查看主日志大小**：
```bash
ls -lh /opt/website_analytics/backend/logs/console.log
```

**清理 30 天前的任务日志**（保留最近一个月）：
```bash
# 先查看将删除哪些目录
find /opt/website_analytics/backend/logs/ -name "task_*" -type d -mtime +30

# 确认后执行删除
find /opt/website_analytics/backend/logs/ -name "task_*" -type d -mtime +30 -exec rm -rf {} + 2>/dev/null
```

**清空主日志（如超过 500MB）**：
```bash
# 备份后清空
cp /opt/website_analytics/backend/logs/console.log /opt/website_analytics/backend/logs/console.log.bak.$(date +%Y%m%d)
> /opt/website_analytics/backend/logs/console.log

# 或直接清空（不备份）
> /opt/website_analytics/backend/logs/console.log
```

---

### 4.3 清理系统日志

**查看系统日志大小**：
```bash
du -sh /var/log/
```

**清理系统日志**（保留最近 7 天）：

```bash
# 清理 journal 日志
```

## 五、服务重启操作

### 5.1 重启后端服务

**先确认服务是如何运行的**（使用 systemd 还是手动）：
```bash
# 检查是否有 systemd 服务
systemctl list-units | grep website

# 或检查进程
ps aux | grep uvicorn
```

**如果是手动运行的**：
```bash
# 1. 找到进程 PID
ps aux | grep uvicorn | grep -v grep

# 2. 结束进程
kill -9 <PID>

# 3. 重新启动（根据实际启动方式）
cd /opt/website_analytics/backend
# uv run uvicorn app.main:app --app-dir api &
```

**如果是 systemd 服务**：
```bash
systemctl restart website-analytics-api
systemctl status website-analytics-api
```

---

### 5.2 重启前端服务

```bash
# 找到进程
ps aux | grep next

# 结束进程后重新启动
cd /opt/website_analytics/frontend
# pnpm start &
# 或
# npm run start &
```

## 六、一键检查脚本

在生产服务器上创建以下脚本：`/opt/website_analytics/daily_check.sh`

```bash
#!/bin/bash
echo "========== Website Analytics 生产服务器每日检查 =========="
echo "检查时间: $(date)"
echo ""

echo "【1/6】磁盘空间检查（告警阈值: 85%）:"
df -h | grep -E "Filesystem|/dev/vda3"
echo ""

echo "【2/6】内存使用检查:"
free -h | grep -E "Mem|Swap"
echo ""

echo "【3/6】CPU 负载检查:"
uptime
echo ""

echo "【4/6】服务进程检查:"
echo "后端服务 (uvicorn):"
ps aux | grep uvicorn | grep -v grep | wc -l | xargs -I {} echo "  进程数: {}"
echo "前端服务 (next/node):"
ps aux | grep -E "next|node" | grep -v grep | wc -l | xargs -I {} echo "  进程数: {}"
echo ""

echo "【5/6】日志目录大小:"
du -sh /opt/website_analytics/backend/logs/
echo ""

echo "【6/6】Playwright 临时文件（/tmp）:"
echo "  临时目录数量: $(ls -d /tmp/playwright_chromiumdev_profile-* 2>/dev/null | wc -l)"
echo "  临时文件总大小: $(du -ch /tmp/playwright_chromiumdev_profile-* 2>/dev/null | tail -1 | cut -f1)"
echo ""

echo "========== 检查完成 =========="
echo ""
echo "如磁盘使用率超过 85%，请执行："
echo "  rm -rf /tmp/playwright_chromiumdev_profile-*"
echo "  rm -rf /tmp/playwright-artifacts-*"
```

**使用方法**：
```bash
chmod +x /opt/website_analytics/daily_check.sh
/opt/website_analytics/daily_check.sh
```

## 七、紧急情况处理

### 7.1 磁盘满了（100%）

**症状**：
- 系统无法写入新文件
- 任务执行失败
- 可能看到 "No space left on device" 错误

**处理步骤**：
```bash
# 1. 紧急清理 Playwright 临时文件
rm -rf /tmp/playwright_chromiumdev_profile-*
rm -rf /tmp/playwright-artifacts-*
rm -rf /tmp/.com.google.Chrome.*

# 2. 清空主日志
> /opt/website_analytics/backend/logs/console.log

# 3. 检查是否恢复
df -h /

# 4. 重启后端服务（因日志无法写入可能已异常）
# systemctl restart website-analytics-api
# 或手动重启
```

---

### 7.2 服务无法访问

**检查步骤**：
```bash
# 1. 检查服务进程
ps aux | grep -E "uvicorn|next" | grep -v grep

# 2. 检查端口监听
ss -tlnp | grep -E "3000|8000"

# 3. 检查防火墙
ufw status
iptables -L -n | grep -E "3000|8000"

# 4. 查看最近错误日志
tail -50 /opt/website_analytics/backend/logs/console.log
```
